This project represents a traditonal 2D platformer with a grappling mechanic that the basic movement of the player. The core mechanics for this game are of course, the character controller,
but more specifically, the grappler. The grappler is used to bridge the gap between jumps that would be impossible for the player alone. The game starts like most games, with a main menu.
The menu features the tutorial for the game, which is a small enviroment for players to get aquainted with the controls of the game. From the menu, you can choose to start the first level of
this game. The level is broken into 3 parts, the first having simple jumps and obstacles, to ease the player into the difficulty, the second featuring a slight increase in difficulty, and the
last part featuring only floating platforms and multiple grapple points in a row. Once you reach the end of the level, the game takes you to a level completion screen. One secondary system
included in this game in additon to the menus is the player life system. You begin with 5 lives and the only way to die is by falling into the void, with each death sending you back to the 
start of the level.

This project has seen significant changes from my orignal GDD, where my orignal goal was to create a puzzle platform/metroidvania, however the game has changed to being mostly a striaght
platformer. Most of the cuts I made, such as the locked door and enemy mechanics, were not added both due to time constraints and that I felt the game would be more enjoyable with more of an
obstacle course vibe rather than an enemy based platformer. The other thing that changed from my orignal GDD was the implicit story that was going to be told. Oringally I wanted the game to
take place in a lab where they accidentally released unknown horrors from another world into the real world, but I deciced to change this both as a result of a gameplay changes as well as
finding it difficult to create/find assets that matched the idea I had for the level. 

Over all, this project served as a way to test my knowledge of Godot as a game engine, and while it isn't perfect and is slightly different from what I had in mind, I am very proud of the
game I created.
